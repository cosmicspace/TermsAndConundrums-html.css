Artistic License 2.0

Definition and details

This license makes an important distinction between the version created by the author, the Standard version and other modified versions. This allows a measure of quality control for the project, since changes to the Standard version can be tested for functionality before becoming the Standard version for distribution. The unmodified Standard version can be patched with the copyright holders’ patches and updates and still be considered the Standard Version: used, copied and distributed as desired. However, once someone else modifies the code, it can no longer be distributed except for the following cases:

1) You provide the changes to the copyright holder, under the Artistic License, and he/she includes those changes into a new Standard version

2) You give the Modified version a new name and it still allows the Standard version to run correctly

3) You provide your source code to others when you distribute it so that they have the option of modifying it, under the same Artistic License or another equivalent license that allows copying, distribution and modification, requires the source code to be available and the same license to be used for derivative works

When this should be used

The main point of this license is to allow the original creator to create a Standard version of the work for quality control, but allow others to use, copy, modify and distribute their changes. These changes can be provided to the original creator to create a new Standard version, which allow improvements to be made on the work by the community. This allows a good, working Standard version to evolve, without the original author making the changes.

The original Artistic License (1.0) has some facets that are unclear legally and this has caused some difficulties in this license being upheld in court. Therefore, Artistic License 1.0 should not be used, in favor of the new revised Artistic License 2.0.

Length or term of license

Length of license corresponds to the work’s copyright. In the USA, this is typically 70 years after the death of the creator, or 120 after it was made by an organization, at which point, the work becomes public domain.

History and shenanigans

The author of the original Artistic License (1.0) is Larry Wall and it was written for Perl, a programming language, under the Perl Foundation.

There was some controversy surrounding the original Artistic License (1.0) in 2007. The Java Model Railroad Interface project (JMRI) used the original Artistic License (1.0) and discovered that KAM, a proprietary company, used Artistic Licensed files in one of their products and removed the copyright notices from the JMRI, which meant they were representing the code as their own work.

The JMRI brought the case to court and the Artistic License was ruled as a contract rather than a copyright license due to two legally ambiguous phrases. This meant that KAM could continue to distribute the software and JMRI could not pursue the higher minimum copyright infringement penalties, instead they could only receive actual damages which are considerably lower as an open source project. This essentially made the Artistic License provide no real protection for project designers.

As a result of this court ruling, a new version of the Artistic License has been created (2.0), with input from attorneys as well as the Perl community. This is a perfect demonstration of what can go wrong if license terms are not perfectly clear. Since the law favors those who did not draft a contract or license agreement, a license needs to be specific and spell out what everyone is allowed to do. Otherwise, the license writers may encounter situations they never expected.

Strangely, Perl, the project for which the Artistic License was written, can be licensed under either the Artistic License or GNU General Public License, version 1 or higher. Depending on which you choose will dictate how derivative works are handled.
