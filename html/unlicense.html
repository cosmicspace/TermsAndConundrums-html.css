The Unlicense

Definition and details

The Unlicense is a public domain dedication. A work released under the Unlicense is dedicated to the public domain to the fullest extent permitted by law, and also comes with an additional lax license that helps cover any cases where the dedication is inadequate. Both public domain works and the lax license provided by the Unlicense are compatible with the GNU GPL.

When this should be used

The Unlicense should be used when you want put something in the public domain, which means you want no claim on the work and anyone can do anything they like with it. There are no requirements or restrictions placed on using the product.

Length or term of license

Forever because it just puts the product in the public domain.

Real world examples

  • Bitcoin.php, a PHP library (and plugins for WP e-Commerce, Ubercart, and VirtueMart) for the Bitcoin peer-to-peer digital currency system.
  • CSS3 Buttons, a simple CSS framework for creating good-looking GitHub style button links.
  • Nightweb, an anonymous P2P social network for Android smartphones and PCs

History and shenanigans

The unlicensed was started on 1/1/2010 as way to get away from copyrighting your product and declaring it in the public domain. It was heavily based how the SQLite open source database project was put in the public domain.

The Free Software Foundation indicates that "Both public domain works and the lax license provided by the Unlicense are compatible with the GNU GPL." However, it recommends the Creative Commons CC0 license over the Unlicense for dedicating software to the public domain since the CC0 license "is more thorough and mature than the Unlicense".
